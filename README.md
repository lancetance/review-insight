AI 用户洞察助手 (AI User Insight Agent)

一个基于大语言模型的评论分析工具,能将海量用户评论自动转化为结构化的洞察报告。针对商品评价与视频社区讨论两类不同性质的评论,采用不同的分析框架,服务于产品决策与内容洞察两种场景。

项目背景

在用户研究、市场分析与内容运营中,常需处理大量用户评论、社区讨论与商品评价。传统的人工阅读归纳耗时且易受主观影响,当评论达到数百上千条时成本显著上升。本工具旨在用大语言模型自动完成从海量反馈 → 主题识别 → 需求/观点提炼 → 洞察生成 → 建议输出的完整分析流程。

核心设计:两类评论,两套框架

本项目的核心思路是:不同性质的评论需要不同的分析方法。

商品评价类——存在明确的"被评价对象",用户在表达对产品的评价,分析服务于卖家与消费者的决策。框架包含:整体概览(好评/差评比例)、用户称赞的优点、用户抱怨的问题(按严重度排序并识别系统性问题)、深层洞察(情绪根源)、给商家的改进建议与给消费者的购买参考。

视频社区讨论类——评论区并非在评价视频本身,而是围绕视频话题展开讨论、共鸣或玩梗。分析服务于内容创作者与研究者。采用"通用底座 + 类型自适应"设计:先根据视频标题与简介/标签判断视频类型,再在五个维度(评论区构成类型、高赞引爆点、社区特色表达/文化、整体情感基调、给创作者的洞察)上动态调整分析侧重。

功能特性


三种输入方式:粘贴文本(支持从网页直接复制的原始评论)、上传 Excel、上传 CSV
两套分析框架:商品评价类 / 视频社区讨论类,由用户选择
类型自适应:视频分析会根据标题与简介/标签判断内容类型,自动调整分析重点
噪音处理:自动识别并忽略界面元素(如"回复""共X条回复"、纯@、emoji、链接),同时保留并利用点赞数、日期等有价值的元数据
结构化报告:输出可读、可下载的洞察报告
访问保护:可设置访问密码,控制 API 调用


技术栈


界面框架:Streamlit
分析引擎:DeepSeek API(OpenAI 兼容接口)
数据处理:pandas、openpyxl
语言:Python
