# AI 用户洞察助手 —— 升级需求说明

## 项目背景
这是一个基于 Streamlit 的评论分析工具,当前已有基础版(上传 CSV → 调用 DeepSeek 分析 → 输出报告),
现在要升级。后端用 DeepSeek API(OpenAI 兼容接口,base_url="https://api.deepseek.com",
模型 deepseek-chat,API Key 从环境变量 DEEPSEEK_API_KEY 读取)。

---

## 升级需求总览

### 一、输入方式(三选一)
界面上让用户选择输入方式:
1. **粘贴文本**:一个大文本框,用户把从网页复制的原始评论(可能带用户名、日期、点赞数等噪音)直接粘进来。
2. **上传 Excel**(.xlsx):用 pandas + openpyxl 读取。
3. **上传 CSV**(.csv):保留原有功能。

上传文件后,让用户选择哪一列是评论文本(沿用现有的列选择逻辑)。

### 二、评论类型(用户选择,决定用哪套分析框架)
界面上让用户选择评论类型:
1. **商品评价类**
2. **视频社区讨论类**

不同类型调用不同的 prompt(见下方框架)。

### 三、视频类型需额外输入标题和简介
当用户选了"视频社区讨论类"时,额外提供两个输入框:**视频标题**、**视频简介**(选填)。
这两项会一起传给模型,让模型先判断视频类型(搞笑/访谈/游戏/知识等),再动态调整分析侧重。

### 四、噪音处理(两类都要)
在 prompt 里明确要求模型:先识别并过滤掉噪音内容——广告、自我推销(如"求点赞我写了几首诗")、
纯@某人、单个 emoji、纯灌水(如只有"绷住""哈哈"),聚焦有信息量的评论再分析。

### 五、参数控制
保留侧边栏的"最多分析多少条评论"滑块,用于控制 API 花费。

---

## 分析框架一:商品评价类

要求模型用中文输出,包含以下部分:

1. **问题/主题归纳(数据驱动,不要预设固定维度)**
   从评论中自行归纳实际出现的主题(如质量、产地与宣传不符、品质下降、包装、气味等,
   完全由数据决定),每个主题给出被提及的大致次数和典型例子。

2. **严重度排序**
   把归纳出的问题按严重程度排序,依据:被提及的频率 + 点赞数/"觉得有用"数(若数据中有)。
   明确指出头号问题。(注意:用户可能只上传差评,因此不要用"正负中比例",改用严重度排序。)

3. **深层痛点与情绪根源**
   挖掘用户为什么愤怒/失望(如被欺骗感、预期落差、信任崩塌),给 3~5 条。

4. **核心洞察**
   指出最致命的问题;识别系统性问题(多人反复提同一点 = 批次/设计缺陷而非个例);
   若数据有日期,留意时间趋势(如品质是否近期下降)。

5. **改进建议**
   针对高优先级问题给 3 条具体可落地的建议。

6. **元数据洞察(可选,数据够规整时才输出)**
   若数据带点赞数、日期、颜色/尺寸等字段,分析:哪个变体差评最集中、问题是否某时间段后爆发。

---

## 分析框架二:视频社区讨论类

**设计原则:通用底座(固定五维度)+ 类型自适应(根据标题/简介动态调整每个维度的侧重)。**

先让模型根据标题和简介判断视频类型与调性,然后在以下五个维度上分析,
并根据视频类型自动调整各维度的轻重(如搞笑视频重点挖"高频梗/二创",访谈类重点挖"观点讨论/情感共鸣",游戏类重点看"吐槽/争议"):

1. **评论区主要构成类型**
   识别评论主要由哪几类组成、各占大致比例。常见类型:玩梗/二创、情感共鸣/个人故事、
   观点讨论、对创作者的反馈/期待、捧场灌水。

2. **高赞引爆点**
   哪几条评论引爆了最多共鸣(高赞)?它们的共同特质是什么?(善用点赞数)

3. **高频梗与流行语**
   评论区反复出现的梗/黑话/流行语是什么(如"绷住""——余华"接龙)?体现社区文化。

4. **整体情感基调**
   评论区的集体情绪氛围是什么(如亢奋玩乐/温暖治愈/引发思考/吐槽抱怨)?

5. **给创作者的洞察**
   观众的反馈、期待、对创作者的态度;可作为选题或方向参考的洞察。

---

## 技术注意事项
- 不要用 HTML <form> 标签。
- API Key 从环境变量读取,不要写死在代码里。
- 保留下载报告功能。
- 代码加中文注释,方便讲解。
- 数据可能很脏(markdown链接、图片地址等),prompt 里要让模型容忍并过滤这些噪音。
